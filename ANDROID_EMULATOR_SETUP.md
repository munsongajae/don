# 모바일 기기/에뮬레이터에서 실행하기

## 1. 개발 서버 시작

### 방법 1: 네트워크 모드로 시작 (권장)
```bash
npm run dev:network
```

또는 직접 실행:
```bash
next dev -H 0.0.0.0
```

이렇게 하면 개발 서버가 모든 네트워크 인터페이스에서 접근 가능합니다.

### 방법 2: 일반 모드로 시작
```bash
npm run dev
```

## 2. 안드로이드 에뮬레이터에서 접속

### 방법 1: 에뮬레이터 전용 주소 사용 (가장 쉬움)
안드로이드 에뮬레이터는 호스트 머신을 `10.0.2.2`로 접근할 수 있습니다.

에뮬레이터의 Chrome 브라우저에서:
```
http://10.0.2.2:3000
```

### 방법 2: 로컬 IP 주소 사용
1. Windows에서 로컬 IP 주소 확인:
   ```bash
   ipconfig
   ```
   IPv4 주소를 확인하세요 (예: `192.168.1.100`)

2. 에뮬레이터의 Chrome 브라우저에서:
   ```
   http://192.168.1.100:3000
   ```
   (실제 IP 주소로 변경)

## 3. 방화벽 설정 (필요시)

Windows 방화벽에서 포트 3000을 허용해야 할 수 있습니다:

1. Windows 설정 → 네트워크 및 인터넷 → Windows 방화벽
2. 고급 설정 → 인바운드 규칙 → 새 규칙
3. 포트 → TCP → 특정 로컬 포트: 3000
4. 연결 허용 → 다음 → 완료

## 4. 문제 해결

### 연결이 안 되는 경우
1. 개발 서버가 `-H 0.0.0.0`로 시작되었는지 확인
2. 방화벽에서 포트 3000이 허용되었는지 확인
3. 에뮬레이터와 호스트가 같은 네트워크에 있는지 확인
4. `10.0.2.2:3000` 대신 로컬 IP 주소를 사용해보세요

### CORS 에러가 발생하는 경우
Next.js 개발 서버는 기본적으로 CORS를 허용하지만, 문제가 있다면 `next.config.js`에서 설정을 확인하세요.

### Hot Reload가 작동하지 않는 경우
에뮬레이터에서는 Hot Reload가 제대로 작동하지 않을 수 있습니다. 수동으로 새로고침하세요.

## 5. iOS 시뮬레이터에서 Safari로 접속 (macOS)

macOS에서 iOS 시뮬레이터를 사용하는 경우:

1. 개발 서버를 `npm run dev:network`로 시작
2. iOS 시뮬레이터 실행
3. Safari 앱 열기
4. 주소창에 입력:
   ```
   http://localhost:3000
   ```
   또는 호스트 IP 주소:
   ```
   http://192.168.1.100:3000
   ```

## 6. 실제 iPhone/iPad에서 Safari로 접속

실제 iOS 기기에서 Safari로 테스트하는 방법:

### Windows에서 실행하는 경우:

1. **개발 서버 시작**
   ```bash
   npm run dev:network
   ```

2. **Windows IP 주소 확인**
   ```bash
   ipconfig
   ```
   IPv4 주소를 확인하세요 (예: `192.168.1.100`)

3. **iPhone/iPad와 Windows가 같은 Wi-Fi 네트워크에 연결되어 있는지 확인**
   - 같은 라우터에 연결되어 있어야 합니다
   - 모바일 데이터가 아닌 Wi-Fi를 사용해야 합니다

4. **iPhone/iPad에서 Safari 열기**
   - Safari 앱 실행
   - 주소창에 입력:
     ```
     http://192.168.1.100:3000
     ```
     (실제 IP 주소로 변경)

5. **방화벽 설정 확인**
   - Windows 방화벽에서 포트 3000이 허용되어 있는지 확인
   - 필요시 인바운드 규칙 추가 (위의 "방화벽 설정" 참고)

### 접속이 안 되는 경우:

1. **방화벽 확인**
   - Windows 방화벽에서 포트 3000 허용
   - 공용 네트워크 프로필에서도 허용

2. **네트워크 확인**
   - iPhone/iPad와 Windows가 같은 Wi-Fi에 연결되어 있는지 확인
   - Wi-Fi 네트워크 이름이 동일한지 확인

3. **IP 주소 재확인**
   ```bash
   ipconfig
   ```
   IPv4 주소가 변경되었을 수 있으므로 다시 확인

4. **개발 서버 로그 확인**
   - 터미널에서 접속 시도가 보이는지 확인
   - 에러 메시지가 있는지 확인

5. **HTTPS 시도 (선택사항)**
   - 일부 네트워크에서는 HTTP가 차단될 수 있습니다
   - Next.js는 기본적으로 HTTP만 지원하므로, HTTPS가 필요하면 별도 설정이 필요합니다

## 7. 물리적 안드로이드 기기에서 테스트

물리적 기기에서 테스트하려면:
1. 개발 서버를 `npm run dev:network`로 시작
2. 호스트와 기기가 같은 Wi-Fi 네트워크에 연결되어 있는지 확인
3. 호스트의 로컬 IP 주소를 사용하여 접속:
   ```
   http://192.168.1.100:3000
   ```

## 참고사항

- 개발 서버는 기본적으로 `localhost:3000`에서 실행됩니다
- 네트워크 모드(`-H 0.0.0.0`)로 실행하면 모든 네트워크 인터페이스에서 접근 가능합니다
- 안드로이드 에뮬레이터는 `10.0.2.2`를 통해 호스트 머신에 접근합니다
- 프로덕션 빌드는 Netlify 등에서 배포된 URL을 사용하세요
